ZevFocus CLI Documentation

A blazing-fast CLI to track deep work, build focus streaks, and share tamper-proof progress.

📦 Installation

Windows:

irm https://2003aditya.github.io/ZevFocus-website/install.ps1 | iex

Or download the .exe directly.

Linux (x86_64):

curl -sSL https://2003aditya.github.io/ZevFocus-website/install.sh | bash

This downloads and installs ZevFocus as zevfocus in your system.

🔐 Login

Register your username (stored locally):

zevfocus login --user "YourName"

🚀 Start a Focus Session

zevfocus start --taskname "Write blog post"

This runs in the background and tracks time actively spent on the system.

🛑 Stop Session

zevfocus stop

Stops the session and prints a summary:

🎯 Session Ended.
                    🕒 Focus Duration: 110 minutes
                    🔥 Streak: 3 days

📊 Summary

Shows total time, sessions, top task, and streaks:

zevfocus summary
🧠 Total Focus Time: 5h 45m
                    📆 Sessions: 4
                    🔥 Streak: 4 days
                    🎯 Top Task: "Debug analytics bug"

🐦 Share to Twitter

Generate tweet-ready output + tamper-proof hash:

zevfocus share

Output example:

🐦 ZevFocus Summary
                    User: YourName
                    🕒 110m deep work
                    🔥 Streak: 3 days

                    Hash: 9f1c2b8e

🔍 Verify a Hash

Visit the verifier at /verify

Paste the hash from the tweet to confirm authenticity.

✅ Hash verified
                    User: YourName

                    Streak: 3 days

⚙️ Advanced Options

  • --debug: Show internal logs
  • --no-share: Prevent hash generation

More config options coming soon.

🧠 How It Works

  • Local data stored at ~/.zevfocus/
  • Session hashes use HMAC for tamper-proof proof
  • Idle detection filters out non-productive time
  • Nothing is ever sent to a server unless shared

🛠 Troubleshooting

  • Command not found? Make sure it's in your PATH
  • Idle not detected? You may not have interacted with keyboard/mouse
  • Want to reset? Delete ~/.zevfocus/db.json

❓ FAQ

  • Is it offline? Yes, fully local by default
  • Is my data stored remotely? No, only local unless you share
  • Can I use it in scripts? Absolutely — automate using shell or cron
